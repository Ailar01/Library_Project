Library System - Web Application

Project Overview

This web application is designed to manage a digital library. Users can register, log in, and interact with books through CRUD (Create, Read, Update, Delete) operations.

Architecture

  • Roles: Admin, Registered User
  • Entities: Users, Books, Transactions
  • Processes: User authentication, book management, transactions (borrowing/returning books)

Database Structure

The application uses a MySQL database with the following main tables:

  • Users Table: Stores user information (id, name, email, password, role, created_at)
  • Books Table: Stores book details (id, title, author, genre, availability)
  • Transactions Table: Logs borrowing/returning events (id, user_id, book_id, borrowed_at, returned_at)

Implementation Approach

The project is built using:

  • Backend: PHP and MySQL (using PDO for security)
  • Frontend: HTML, CSS, Bootstrap
  • Security: Password hashing, form validation, SQL injection protection
